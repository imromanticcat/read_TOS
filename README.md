# 약관을-읽어주세요: 이용자를 보호하는 약관 분석 경량화 모델

## 0. 환경 구조
VSCode Remote Connection <=> Github Codespace <=> Google Colab Kernel

## 1. AI 서비스 개요 (Service Overview)

  * **서비스 명칭:** 약관을 읽어주세요
  * **핵심 아이디어:** 자연어 처리(NLP) 기반의 경량 텍스트 분류 모델을 활용하여 기업 이용 약관의 개별 조항에서 '기업이 사용자에게 악의적으로 손해를 입힐 수 있는 악용 가능성'이 있는 취약점을 자동으로 판별하고 사용자에게 직관적으로 경고하는 Chrome Extension 서비스.

## 2. 문제 제기 및 제안 배경 (Problem Statement & Background)

오늘날 개인정보 이용 동의서는 복잡하고 방대하여 일반 이용자가 내용을 모두 이해하는 것이 사실상 불가능합니다. 이는 다음과 같은 문제를 야기합니다.

  * **정보 불균형 및 시간 비용:** 미국 연구(Cranor와 McDonald)에 따르면, 평균 인터넷 사용자가 1년에 방문하는 웹사이트의 개인정보 처리방침을 모두 읽는 데 약 244시간(근무일 기준 약 30.5일)이 소요되어, 이용자가 자신의 프라이버시를 지키기 위해 막대한 '시간'이라는 사회적 기회비용을 지불하고 있습니다.
  * **형식적 동의 관행:** 국내 설문조사 결과, 인터넷 이용자의 약 72%가 약관을 거의 읽지 않고 '묻지마 동의'를 수행하며, 이는 중요 정보 처리 절차를 제대로 파악하지 못한 채 민감한 정보 제공 및 해외 이전 등이 이루어질 위험을 높입니다.
  * **실제 악용 사례:** 청소년 고민상담 앱 '나쁜 기억 지우개'의 고민 글 통계 데이터 판매 논란, 중국 앱 '콰이(快手)'의 무단 영상 활용 등 이용자가 이해하지 못한 동의 조항이 기업에 의해 악용되는 구체적인 사례들이 존재합니다.

이러한 문제를 해결하고 이용자의 실질적인 자기결정권을 보장하기 위해, AI 기반으로 약관을 분석하고 위험을 직관적으로 알려주는 지원 시스템이 필요합니다.

## 3. 세부 내용 및 기술 스택 (Technical Details & Stack)

| 구분             | 내용                                                                                                                    |
| :------------: | :-------------------------------------------------------------------------------------------------------------------: |
| **모델 구조 및 목표** | 자연어 처리(NLP) 기반 **경량 텍스트 분류 모델** 개발. 약관의 개별 문장(조항)을 입력받아 '기업의 악용 가능성이 있는 취약 조항', '일반 조항' 등의 **다중 클래스(MultiClass)로 분류** |
| **기반 모델**      | 학습 및 추론 속도, 경량화를 고려하여 **DistilBERT** 선정. Legal\_Specific\_KoLLM 등의 법률 특화 모델 접근 방식 참고                                  |
| **학습 데이터**     | **TOS\_dataset**, **Lex-Glue - Unfair TOS dataset**을 활용한 모델 파인튜닝(Fine-tuning)                                         |
| **배포 환경**      | **Chrome Extension(크롬 확장 프로그램)** 형태로 배포                                                                               |
| **모델 최적화**     | 서버 부하 및 외부 전송 없이 즉각적인 분석을 위해 **사용자 로컬(Local) 환경 구동** 설계. **ONNX 변환** 및 **양자화(Quantization)** 적용을 통한 추론 속도 향상 및 용량 최소화 |
| **서비스 기능**     | 사용자가 접속한 웹페이지의 이용 약관 및 개인정보 처리방침 텍스트를 실시간으로 분석. 위험 조항에 대해 **하이라이팅, 경고 아이콘 표시 등 직관적인 시각화** 제공                          |

## 4. 기대 효과 (Expected Outcomes)

| 구분          | 기대 효과                                                   |
| :---------: | :-----------------------------------------------------: |
| **사회적 가치**  | 일반 사용자의 정보 불평등 해소 및 소비자 권익 선제적 보호.                      |
| **경제적 가치**  | 투명한 거래 환경 조성 유도 및 약관 관련 분쟁 비용 감소 효과 기대.                 |
